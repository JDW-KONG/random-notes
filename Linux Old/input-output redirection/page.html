Three Things to Remember
- stdin is the keyboard
- stdout is the screen
- stderr is reserved for error outputs

- the most common use of output redirection is collecting the output oof a command into a file

- ls -al > listing
- stores output from ls in listing
- use >> to append

- input redirection
  • - redirect where your system is getting input from
  • - use the sort command
    • - sort < listing
      • - passes listing file into sort command
    • - sort < listing > sorted_list
      • - stores the output of sort in sorted_list
- pipes are used to take the output of one command and feed it into another
- give any command scrolling output by piping it through less

- we can also redirect errors
  • - myscript.sh 2> errors
    • - the 2 tells it to redirect stderr instead of stdout
    • - normal output will still be displayed on the screen
    • - errors will be written to a file named errors
  • - myscript.sh > allout.txt 2>&1
    • - redirects all output and errors to a file
    • - prints all errors to stdout
