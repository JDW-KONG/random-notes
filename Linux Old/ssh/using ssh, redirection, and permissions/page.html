- set up a new server for a dev to use
- needs to be able to connect via ssh without password

- copy tar files from one server to another
- send all output from the extraction to /home/dev/tar-output.log

- make sure new files have to correct permissions

- verify permissions on a script


- enable SSH to connect without password
- generate ssh keys
- ssh-keygen

- copy ssh keys to remote server
- ssh-copy-id [remote server ip]

- log in to remote server
- ssh [username]@[remote server ip]

- copy all tar files from local server to remote server
- use scp to copy files
- scp [filename] [remote server ip]

- connect to remote server
- ssh [username]@[remote server ip]

- verify that files have transferred successfully
- ll

- extract files making sure output is redirected
- tar -xvf [filename] >> [output file].log

- set umask so that new files are only readable and writable by owner
- umask 066

- verify that script is executable and run it
- check permissions on file
- ls -l deploy.sh

- add execute bit
- chmod +x deploy.sh

- run the file
- ./deploy.sh
