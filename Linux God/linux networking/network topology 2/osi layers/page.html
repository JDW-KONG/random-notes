OSI Layers

OSI
- open systems interconnection model
- conceptual model that standardizes network communication
- the goal is the interoperability of diverse communcation systems with standard protocols
- originally seven layers
- the model partitions a communication system into abstract layers


Application Layer
- the interface permitting the user to send and receive daata through clients and applications
- HTTP, POP3, SMTP, DNS, SOAP
- verify that application is functional and DNS is correct

Presentation Layer
- converts the request from the application into a universal format
- TLS, SSL, MIME, JPG, GIF

Session Layer
- permits the ability to open, close, and manage a session between the process and response
- LDAP, NetBIOS, PPTP, RPC, SMB, SSL

Transport Layer
- defines how data will be sent, providing validation and security
- TCP, UDP, SPX, iSCSI
- verify blocked (or damaged) ports, firewalls, and QoS

Network Layer
- looks for the best path to reach the destination [ IP ]
- IP, IPSec, ICMP, RIP
- verify addressing and routing configuration, bandwidth, and authentication

Data Link Layer
- handles communication between adjacent nodes
- Ethernet, Frame Relay, IEEE 802.11 (WiFi), PPP, VLAN, MAC
- verify switch and VLAN configuration as well as MAC addressing and IP conflicts

Physical Layer
- handles bit level transmission between network nodes
- Ethernet physical, IEEE 1394 (firewire), infrared
- verify physical components
