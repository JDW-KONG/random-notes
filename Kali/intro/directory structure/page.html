- apt-get install tree
- prints directory structure

- apt-get install mlocate
- helps locate files

- we need to understand where everything is
- everything is a file, even a directory

- bin: where binaries are stored when software is installed
- executable commands live here

- boot: do not touch unless you know what you are doing
- may break installation

- dev: lists currently connected devices and drivers

- etc: everything to configure
- stores config files

- home: where user's home directories are stored

- lib & lib64: stores libraries

- lost & found: files that cannot be fully purged from thr system yet

- media: usb devices and other storage

- mnt: where volumes are manually mounted

- opt: where optional programs are installed

- proc: contains info about the running system
- cpu, ram, etc.

- root: the home folder of the root user
- do not store anything here

- run; used to store data

- sbin: sudo bin - stores sudo level binaries

- srv: contains data for servers

- usr: more libraries, used to be the user's home directory folder

- var: log files
- /var/log
- VERY IMPORTANT

- tmp: stores temporary files

- sys: virtual directory that contains information about connected devices
- DO NOT TOUCH
