.bash_profile or .profile
- designed to execute scripts when you log in
  • - when entering username and password
- runs exports and aliases as part of the login process
  • - runs these programs as if you had opened a new terminal
  • - can be executed manually to apply updated settings
- typically includes .bashrc
.bashrc
- executed before the command prompt loads
- or when bash is typed into the shell
- can set the path here


.bash_history
- the history of what you have typed into a shell


.bash_logout
- the last thing executed when you enter the exit command or logout


what makes a file a shell script?
- contains commands

- defined by the fact that it starts a shell and executes commands within that shell

- file starts with #!/bin/bash

- file does not need an extension


displaying environment variables in a script
- use a dollar sign and the environment variable name
  • - $HOME

using variables on the command line
- FIRSTNAME="Dogma"
- export FIRSTNAME
- echo "Your first name is: $FIRSTNAME"

- export SEARCHSITE="https://www.google.com"
- export SEARCHSITE="http://www.bing.com"
- updates the variables value


setting and using variables in scripts
- VARIABLE="value"


comment types and structures
- precede comments with a #
- # this is a comment


command substitution
- use backticks around commands


exit status
- $?
- two primary statuses:
  • - 0: ok
  • - ANYTHING ELSE: an error has occurred'
  • - set -e: exits on error
- helps us prevent unintended consequences
arithmetic operations
- expr
  • - evaluates expressions
  • - include spaces between operator
  • - escape illegal chars with \
global and local environment variables
- printenv
- env
  • - env | grep thing

special characters
- escape chars with \
- single quotes print literally, no variable conversion
- `command` executes command and returns output to string


using /dev/null
- actually a device
- considered the black hole of your system
- anything sent there vanishes
- >> /dev/null


the read statement
- allows us to capture info from users
- read VARIABLENAME


shell expansion
- the ability to use shortcuts
- use ~ to represent home directory

- declaring a variable and exporting
- echo "${VARNAME:=value}"


types of variables
- the declare keyword - explicit variables
- declare -p MYVAR
- checks variable info
- declare a variable a specific type
- declare -i NEWVAR=10
- marks this variable as an integer
- declare +i NEWVAR
- changes variable type to all

- declare -r READONLY
- creates a readonly variable

- readonly MYREADONLY="This string"
- alternate way of creating readonly variable


arrays
- an indexed list
- MYARRAY=("First" "Second" "Third")

- echoing an array will only print the first value

- SELECTING AN ITEM FROM AN ARRAY
  • - ${MYARRAY[0]}
  • - ${MYARRAY[*]}
- ADDING TO AN ARRAY
- MYARRAY[3]="Fourth"

- SELECTING ALL ITEMS FROM AN ARRAY
-${MYARRAY[@]}
