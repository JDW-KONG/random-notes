- encoding bits into packets prior to transmission
- decoding packets back into bits at the destination

Ethernet: Principles, Frames, and Headers

- relatively inexpensive, reasonably fast

- commerically introduced in 1980 and first standardised in 1983 as IEEE 802.3
- has been refined to be faster and more stable over long distances

- ethernet design principles
- multiple computers can send data at any time
- collision handling: carrier sense multiple acees - collision detection (CSMA/CD)
    • - bith computers will stop sending and pick a random time to retransmit data
- cable types
  • - 10Base2, 10Base5
  • - old
  • - 10BaseT, 100BaseT, 1000BaseT
  • - 10BaseF
  • - fiber cables
  • - initial number is the speed in Mb
- ethernet packets at the data link layer send frames

- ethernet packet at physical layer
- 8 bytes
- contains destination mac address and start frame delimiter

- ethernet type II frames
  • - each ethernet frame header contains:
  • - 14 bytes
    • - destination mac address
    • - source mac address
    • - ethernet type
  • - ethernet frame also contains data to be sent
  • - 46 - 1500 bytes
  • - IP, ARP, Payload, etc.
  • - also includes a frame check sequence
  • - 4 bytes
ARP: Address Resolution Protocol
- a network layer protocol used for mapping an ipv4 address to a physical address

- one computer broadcasts an arp request asking for destination computer's mac address
  • - each system on the network segment receives the request
  • - only the computer with the correct mac address will respond
  • - each machine uses arp requests to build an arp table
- arp tables
- computers cannot send an arp request broadcast each time they need a mac address
  • - they store the ip address and corresponding mac address in a table for future reference
  • - entries are updated every 15 to 20 methods
  • - look into the arp table of a system that you are scanning to see other devices on the network


VLANs - Virtual Local Area Networks
- a lan is a local area network
- group of devices that share a common communication line
- lan connections are used to share resources locally

- virtual lan is a group of hosts that communicate like a lan, even if they are not on the same switch
  • - partitioned and isolated in a computer network at the data link layer
  • - applies tags to network packets
  • - creates the appearance of traffic on a single network
- why use VLANS
- groups with sensitive data are separated from the rest of the network
- cost reduction, less need for network upgrades
- higher performance
- reduces the number of router hip and increases apparent bandwidth
  • - broadcast storm mitigation
    • - automatically reduces network broadcasts
  • - improved IT staff efficiency
  • - makes it easier to manage the network
  • - simpler company management
  • - can have VLANs dedicated to specific types of employees

WLANS

- wireless networks connect nodes using radio communication

- WLANs link devices over a short distance using an access point

- uses the IEEE 802.11 family of communications
  • - most widely used wireless communication standard
  • - 802.11 b and c are most popular
  • - 802.11 ac is the newest standard
  • - includes wider channels and 5GHz band
- wireless networks are identified using SSIDs
  • - service set identifiers
  • - SSID
    • - generic term, referring to the wireless network name
  • - BSSID
  • - basic SSID - the MAC address of the AP
  • - ESSID
  • - extended SSID, a name applied to one or more APs providing the same service to access a wired lan
    • - in some deployments, each AP has its own unique ESSID
    • - in others, a single ESSID is shared across multiple access points to support roaming
- most significant wireless network attacks
- denial of service
    • - blocking legitimate users from accessing the network
  • - rogue access points
  • - unauthorized access points set up to bypass wireless security
  • - misconfiguration
    • - access points not configure appropriately
  • - traffic capture/intercept
  • - sniffing the network
  • - crypto attacks
    • - weak wireless encryption protocols
  • - client-duping
  • - set up a bogus access point and access data
