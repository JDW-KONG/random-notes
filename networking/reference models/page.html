What is a Protocol?

- the internet
- an interconnected system of computer networks
- a network comprised of networks
- uses TCP/IP as a standard protocol
- millions of private networks linked by a broad array of technologies

- originated in the 1960s
- developed by the US government

- started as the ARPAnet
- backbone for interconnection of regional academic and military networks
  • - more than 4.1 billion people use the internet
- a protocol is a set of rules that govern communication over the internet
  • - protocols standardize network communications
  • - uses the right protocol for the right program
  • - protocols are to computers what language is to humans
  • - for two devices on a network to communicat, they must use the same protocol
- IP: Internet Protocol
  • - how data packets move through a network
  • - protocols ensure that all machines are speaking the same language
  • - standardizes how devices route ip packets
    • - forwarding IP packets from source to destination IP addresses
  • - IP addresses are used to identify a device on a network
  • - this IP address makes the computer accessible to other devices


Reference Models

- conceptual frameworks for understanding relationships
- guides users so that programs they create can communicate


OSI Reference Model

- Open System Interconnection
- reference model for how applications can communicate over a network

- OSI is rarely actually implemented
- model does not accurately represent modern communcations

- the process between two endpoints can be divided into seven unique layers
- data flows up and down the layers during the communcation process

- layer 7: application layer
  • - the most known protocols
- example: HTTP, Telnet, FTP, SMTP, TFTP, DNS, SNMP
  • - layers 5 - 6 are often grouped together as the application layer
- layer at  which communication partners are identified
- apps with user interaction
- the set of services an application can make use of

- layer 6: presentation layer
- usuallly part of an OS
- different formats: ASCII, MP3, JPG
- converts incoming and outgoing data from one format to another
- ex: cleartext to encrypted text

- layer 5: session layer
- sets up, coordinates, and terminates conversations
- sessions and sockets: SQL, RPC
- authentication and reconnection after interruption
- TCP and UDP provide these services for most applications

- layer 4: transport layer
- example: TCP, UDP
- manages packetization of data
  • - checks for errors when packet arrives
- flow and error control handled by: TCP, UDP

- layer 3: network layer
- example: IP
  • - addresses hosts, encapsulates data into packets
- end to end communcation and virtual addresses are handled by IP
- sends data to the right destination and receives packets as well
  • - IP is the network layer for the internet

- layer 2: data link layer
  • - example: 802.3 ethernet, 802.4 token bus, 802.11 wireless lan, fddi
- sets links on the physical network and puts packets into frames
- two sublayers
- media access control layer
- logical link layer
- ethernet is the main data link layer in use

- layer 1: physical layer
  • - example: token ring, 10Base-T, 100Base-T
- transports bits across the network in electrical, optical, or radio level
- provides hardware means of sending and receiving data

- the data unit transferred between two endpoints has a different name on each layer
- layers 5, 6, and 7 transmit data
- layer transmits segments and datagrams
- layer 3 transmits packets
  • - layer 2 transmits frames
  • - packets at layer one and layer two are managed by the network interface of your device

  • - packets at layers 3 -7 are managed by the CPU and OS
  • - layer 3 and layer 4 packets are managed by the OS
  • - packets at layer 5 -7 are managed by their related application or service

OSI vs TCI/IP

- TCP model is similar to the OSI model
- represents the real world implementation of networks

- application, presentation, and session layers are combined into application layer in TCP/IP model

- physical adn data link layers are combined into the data link layer in the TCP/IP model
