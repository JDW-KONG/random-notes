IPv4 Subnetting: Classful Networks

- octets of the ip address are broken down to provide an addressing scheme for small and large networks

- the IP address class can be determined

- class A address
  • - first octet is the network portion
  • - major network address of 1.0.0.0 to 127.255.255.255
  • - used for networks that have more than 65535 hosts
- class B address
  • - first two octets are the networ address
  • - major network address of 128.0.0.0 to 191.255.255.255
  • - octets 3 and 4 are for local subnets and hosts
  • - used for networks that have between 256 and 65535 hosts
- class C address
- first three octers are the network portion
- major network address of 192.0.0.0 to 223.255.255.255
- the fourth octet is for local subnets and hosts
- good for networks with less than 254 hosts


IPv4 Subnetting: Subnet Mask

- subnet mask identifes the network portion and the host portion

- default masks
- class A: 255.0.0.0
- class B: 255.255.0.0
- class C: 255.255.255.0

- a subnet mask can be customized to divide a network into many small portions

- convert the address and mask into binary numbers
- identification of the network and host id is a lot easier
- address bits with corresponding maskbits  set to 1 represent the network id
- address bits with corresponding mask bits set to 0 represent the host id

- IP: 131.107.1.2
- Net Mask: 255.255.0.0
  • - if 131.07's corresponding mask bits equal 1, it is the network id
  • - if .1.2's corresponding mask bits equal 0 in, it is the host id
  • - all devices on this segment share the same network id and subnet mask
  • - subnet mask can be calculated to divide a network into many smaller portions

- network id may be expressed in CIDR notation
- classless inter domain routing
- 131.107.0.0/16
- the first address of the network is 121.107.0.0
- the first 16bits (131.107) are for the network prefix
- the last 16bits (0.0) are for subnets and hosts

- the first address of a network, followed by a slash character and ending with the bit length of the prefix


IPv4 Subnetting: Understanding

- subnetting allows you to create logical networks that exist within a single class A, B, or C network
- if you ddo not subnet, you are only able to use one network in your class A, B, or C network

- designate high order bits from the host as part of the network prefix
- our ip address is: 192.168.2.130 = 11000000.10101000.00000010.10000010
- our subnet mask is 255.255.255.192 = 11111111.11111111.11111111.11000000
  • - use the first two bits of our host portion as part of the new network prefix
    • - we want our host id 's corresponding mask to equal zero
    • - 192.168.2.130 = [11000000.10101000.00000010.10]000010
    • - 255.255.255.192 = [11111111.11111111.11111111.11]000000
      • - we have given the first two bits of of host portion to the network prefix
      • - the host id's corresponding network mask is now equal to zero
    • - we can now use the last 6 bits of our hosts
    • - the first 26 bits will be used for the network prefix
    • - new subnets in CIDR notation:
      • - 192.168.2.0/26
      • - 192.168.2.64/26
      • - 192.168.2.128/26
      • - 192.168.2.192/26
    • - there are a possible 2^6 IP addresses in each subnet
    • - 2 ^ number of 0s
  • - 000000 and 111111 are reserved addresses that cannot be used
      • - the total number of available addresses on each subnet can be calculated with the following:
      • - 2 ^ number_of_available_bits - 2
      •  
      • - 2 ^ 6 = 64 , 64 - 2 = 62
      • - 62 possible IP addresses
          • - 192.168.2.1 - 192.168.2.62
          • - 192.168.2.65 - 192.168.2.126
          • - 192.168.2.129 - 192.168.2.190
          • - 192.168.2.193 - 192.168.2.254
          • - 192.168.2.129 - 192.168.2.190 is our new subnet range
