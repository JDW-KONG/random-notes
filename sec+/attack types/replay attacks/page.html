Replay Attack
-a man in the middle attack that intercepts data but replays it at a later date
- kerberos, a microsoft authentication protocol can prevent this as each entry has updated sequence numbers and timestamps
- expects data in a certain sequence and rejects the data if it is out of order

- useful information is transmitted over the network
- a crafty hacker will take advantage of this

- need access to the raw network data
- network tap, arp poisoning, malware on victim computer

- the gathered information may help the bad guy
- replay the data to appear as someone else

- this is not a man in the middle attack
- the actual replay doesn't require the original workstation

- legitimate user send auth request to server
- bad guy also receives a copy of this data
- bad guy later replays data captured from user to access user's account

- avoid this type of replay attack with a salt or encryption
- use a session ID with the password hash to create a unique authentication hash each time
