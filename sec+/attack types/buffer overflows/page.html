Buffer Overflows
- occurs when a program tries to store more data than it can hold in a temporary memory storage area

- writing outside the allocated memory into the adjacent memory can:
- corrupt the data
- crash the program
- cause the execution of malicious code

- overwriting a buffer of memory
- data spills over into other memory areas

- developers need to perform bounds checking
- attackers spend a lot of time looking for openings

- not a simple exploit
- takes time to avoid crashing things
- takes time to make it do what you want

- a really useful buffer overflow is repeatable
- which means that all systems are pwned


Integer Overflow
- occurs when the result of an arithmetic operation exceeds the maximum size of the integer type used to store it
- similar to a buffer overflow but uses numerical values
