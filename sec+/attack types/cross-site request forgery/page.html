Cross-site Request Forgery
- cross site request forgery
- when a user clicks on a link to a legitimate website where embedded programming is executed
- can be used to:
    • - modify firewall settings
    • - post unauthorizzed data on a forum
    • - conduct fraudulent financial transactions

- one-click attack, session riding
- XSRF, CSRF

- takes advantage of the trust that a web application has for the user
- the website trusts your browser

- significant web application development oversight
- the application should have anti-forgery techniques added
- usually a cryptographic token to prevent a forgery

- three devices: attacker, victim, server
- bad guy creates a request
- request is sent as a hyperlink to a user who may already be logged into the site
- visitor clicks the link and unknowingly sends the request to the site
- website validates attackers request
