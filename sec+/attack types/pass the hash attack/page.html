Pass the Hash Attack

- attacker first obtains the hashes from a targeted system that is using an NT Lan Manager (NTLM) using any number of hash dumping tools

- a pass the hash tool is then used to place the obtained hashes on a local security authority subsystem service

- a windows based authentication system believes that the attacker is a legit user
- windows automatically provides the required login credentials

- can all be accomplished withoout the original password
- account lockout cannot be enforced because this attack is operating on a lower layer

- you must disable NTLM or enable kerberos to prevent the pass the hash attack
