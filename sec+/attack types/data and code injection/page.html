Code Injection
- adding your own info into a data stream
- the end user should not be able to add code to the program

- enabled because of bad programming
- the application should properly handle input and output

- there are many data types that can be injected into applications


SQL Injection
- when a SQL command is modified to gain information from the database by ending the statement with 1=1
- structured query language
- the most common relational database management system language

- modifying SQL requests
- application should not allow this

- stored procedure
- a prewritten SQL script
- cannot be modified by attackers


XML Injection and LDAP Injection
- XML
- extensible markup language
- a set of rules for data transfer and storage

- XML Injection
- modifying XML requests
- a good application will validate


Input Validation
- when an application only accepts data in the correct format within a range of minimum and maximum values

- can prevent SQL Injection, buffer overflow, and integer overflow attacks
