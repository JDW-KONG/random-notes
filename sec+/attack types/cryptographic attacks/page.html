Cryptographic Attacks

- you've encrypted data and sent it to another person
- is it really secure
- how do you know

- the bad guy doesn't have the key, so he breaks the encryption

- there are many potential cryptographic shortcomings


Account Lockout
- the number of attempts a user can make to insert their password before being locked out of the system
- can stop both dictionary and brute force attacks


Minimum Number of Characters
- the best way to prevent the password from being cracked is salting or key stretching
- randomized characters are inserted into the password
- this will prevent a dictionary attack, will only slow down a brute force attack


Login Errors
- inserting an incorrect password


Weak Implementations
- using the most common passwords
- a low number of characters
- simple passwords
- default passwords for devices

- weak encryption
- one weak link breaks the entire chain

- 802.11 WEP
- the RC4 key can be recovered by gathering enough packets
- the algorithm didn't sufficiently protect the key

- DES - data encryption standard
- relatively small 56-bit keys
- modern systems can brute force this pretty quickly


Birthday Attack
- in a classroom of 23 students, what is the chance of two students having the same birthday?
- 23 students - about 50%
- for a class of 30 the chance is about 70%

- in the digital world this is known as a hash collision
- a hash collision is the same hash value for two different plaintexts
- attackers try to find a collision through brute force

- the attacker will generate multiple versions of plaintext to match the hashes
- protect yourself with a large hash output size

- if we store passwords as hashes, all passwords that are the same will produce the same hash if we use the same hashing algorithm
- look for collisions in hashes
- if two hashes with the same value are found, the passwords are the same

- digital signatures are susceptible to a birthday attack


Known Plaintext Attacks
- attacker has both the plaintext and the encrypted data
- if you know the original plaintext, you may be able to find a wedge that is revealed in the ciphertext
- the known plaintext is the crib

- WWI enigma cipher
- easier to break if you knew some plaintext
- daily weather report (wetter)
- numbers were common (eins)
- royal air force would "seed" the north sea with mines
- future messages woul reference the harbor name


Rainbow Tables
- an optimized, pre-built set of hashes
  • - lists of precomputed passwords with a corresponding hash
- these tables speed up the cracking of passwords that have been hashed
- doesn't need to contain every hash
- the calculations have already been done

- remarkable speed increase
- especially with longer password lengths

- need different tables for different hashing methods
- windows is different than mysql

- rainbow tables won't work with salted hashes
- additional random value added to the original hash


Dictionary Attacks
- people use common words as passwords
- you can find them in the dictionary

- if you're using brute force, you should start with the easy ones
- password, ninja, football

- many common wordlists are available on the net
- some are customized by language or line of work

- this type of attack will catch the low hanging fruit
- you'll need smarter attacks for smarter people

- using all the words in a dictionary to crack a password


Brute Force
- the password is the key
- secret phrase
- stored hash

- brute force attacks - online
- keep trying the login process
- very slow
- most accounts will lock out after a number of failed attempts

- brute force the hash - offline
- obtain the list of users and hashes
- calculate a password hash, compare it to the stored hash
- large computational resource requirement

- runs through all of the different combinations of letters and characters and will eventually crack a given password
- online mode: the attacker must use the same login interface as the user's application

- offline mode: requires the attacker to steal the password file first
- this will give them unlimited attempts at guessing the password


The Password File
- different across operating systems
- different has methods


Collisions
- hash digests are supposed to be unique
- different input data should never create the same hash

- MD5 hash
- message digest algorithm  5
- first published in 1992
- collisions identified in 1996

- december 2008: researchers created CA cert that appeared legit when MD5 is checked
- built other certificates that appeared legit and issued by rapidssl


Collision Attack
- an attack on a cryptographic hash that tries to find two inputs producing the same hash value
- known as a hash collision


Salting Passwords
  • - where a random set of characters is inserted into or appended to a password hash
  • - prevents duplicate passwords from being stored
  • - prevents rainbow table and collision attacks


Downgrade Attacks
- instead of using perfectly good encryption, use something that's not so great
- force the systems to downgrade their security

- 1995 - SSL/TLS vulnerability - FREAK - Factoring RSA Export Keys
- public key pairs can be limited to 512 bits or less
- 1990 U.S. cryptography export regulations

- weak keys could be forced during the ssl handshake
  • - modern systems can easily brute force the small keys

- vulnerability was eventually patched


Replay Attacks
- some cryptographic algorithms are more susceptible than others to a replay attack
- a hash with no salt, no session ID tracking, no encryption

- replay countermeasure may be part of the cryptography
- kerberos and kerberos derivatives include time stamps
- anything after the time to live (TTL) is discarded


Hybrid Attack
- a combination of both a dictionary attack and a brute force attack
