Rogue Access Points and Evil Twins


Rogue Access Points
- someone entering your network and connecting their own open system authentication wireless access point
- use an 802.1x managed switch
- authenticates the device before it can be functional

- a significant potential backdoor
- huge security concerns

- very easy to plug in a wireless AP
- or enable wireless sharing in your OS

- schedule a periodic survey
- walk around youe building/campus
- use third-party tools/wifi pineapple

- consider using 802.1X (network access control)
- users must authenticate, regardless of the connection type


Wireless Evil Twins
- a wireless access point with no security that is made to look like a legit access point
- when a user connects to it, all of their traffic is scanned and the user is unaware

- buy a wireless access point

- configure it exactly the same way as an existing network
- same SSID and security settings

- overpower the existing access points
- may not require the same physical location

- wireless hotspots are easy to fool
- and they are also wide open

- always encrypt your communication
- use https and a VPN
