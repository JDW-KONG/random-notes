Wireless Disassociation Attacks
- a significant wireless denial of service attack
- refers to when a user is using wifi and then all of a sudden they are disconnected and can no longer see the WAP
- similar to a DDoS attack

- the only way to prevent this attack is to use a wired connection


802.11 Management Frames
- 802.11 includes a number of management features
- frames make everything work
- you never see them

- important for the operation of 802.11 wireless
- handles finding access points, manages QoS, associates/disassociates with an access point, etc.

- original wireless standards did not add protection for management frames
- data is sent in the clear
- no authentication or validation is performed


Protecting Against Disassociation
- IEEE has already addressed the problem
- 802.11w - July 2014

- some of the important management frames are encrypted
- disassociate, deauthenticate, channel switch announcements, etc.

- not everything is encrypted
- beacons, probes, authentication, association

- 802.11w is required for 802.11ac compliance
- this will roll out going forward
