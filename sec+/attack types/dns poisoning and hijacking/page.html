DNS Poisoning and Hijacking
- DNS poisoning is the process of putting bad entries into the DNS cache
- diverts requests to a fraudulent website that looks like the real website

- modify the DNS server
- requires some crafty hacking
- manipulate info within the DNS server to send users to the wrong sites

- modify the client host file
- the host file takes precent over DNS queries

- send a fake response to a valid DNS request
- requires a redirection of the original request or resulting response


DNS Spoofing/Poisoning
- user 1 makes a request to the DNS server
- bad guy takes control of the server and modifies DNS values
- user 2 makes a request and is redirected to a site of the attackers choosing

- dns poisoning
- DNS records are first checked on the DNS cache on the local machine
- the hosts file is checked afterwards, and then the DNS server

- DNSSEC
- encrypts DNS traffic
- designed to prevent DNS spoofing attacks, DNS cache poisoning, and DNS traffic capture
- produces RRSIG records for each entry in the DNS database


Domain Hijacking
- when someone tries to change the domain registration of a domain with internet authorities so that they can control it for profit.
- attacker re-registers the domain of a well known company
- gains access to the control panel with the original domain's company
- take out a hosting package with another hosting company
- copy all files from old hosting to new hosting
- point DNS records to new hosting to take sales from old host

- get access to the domain registration and you have control where the traffic flows
- you don't need to touch the actual servers
- determines the DNS names and DNS IP addresses

- many ways to get into the account
- brute force
- social engineer the password
- gain access to the email address that manages the account
